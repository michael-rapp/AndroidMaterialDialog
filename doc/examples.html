Examples

This is example provides a quick overview about how to use the library "AndroidMaterialDialog". This includes the necessary setup steps, as well as source code examples, which demonstrate the use of the library.

Project setup

Before the library "AndroidMaterialDialog" can be used, its source code must be retrieved. The latest release can be downloaded as a zip archive via the direct link https://sourceforge.net/p/androidmaterialdialog/files. Alternatively, the Mercurial repository, which hosts the project's source code, can be cloned by using the following command line command:

hg clone http://hg.code.sf.net/p/androidmaterialdialog/code androidmaterialdialog-code

To use the library within an Android project, the downloaded source code has to be imported into the workspace of the appropriate IDE. When using Eclipse, the source code can be added as a new project by using the option "File" -> "Import..." -> "Android" -> "Existing Android Code Into Workspace".

To be able to use the newly imported library project from within another project, a reference to the library project has to be created. This can be established in the Eclipse IDE by right-clicking the dependent project and choosing the option "Properties". Then the group "Android" at the left of the recently opened window has to be chosen. Finally, the button "Add..." has to be clicked to add the library project as a new dependency. For further information about how to reference an Android library project, refer to this article of the official Android developer documentation.

As the library depends on the AppCompat v7 revision 21 support library in order to provide Android 5.0-features on devices with an API level less than 21, this support library or a newer one has to be added to the library's workspace as well. A detailed explaination about how to import this support library can be found in this article of the Android developer documentation.

Creating a typical alert dialog

The code below shows how to create and show a typical alert dialog, which may be used to asks the user for confirmation of a specific action. The button contains a title and a message and can be closed by the user by either using a "OK" button or a "Cancel" button. The this parameter, which is passed to the builder's constructor in the example below, must be a Context, e.g. an Activity. The null parameters, which are passed to the setPositiveButton- and setNegativeButton-methods can be replaced by instances of the type DialogInterface.OnClickListener in order to execute some code when the user closes the dialog by clicking the appropriate button.

MaterialDialogBuilder dialogBuilder = new MaterialDialogBuilder(this);
dialogBuilder.setTitle(R.string.dialog_title);
dialogBuilder.setMessage(R.string.dialog_message);
dialogBuilder.setTitle(R.string.dialog_title);
dialogBuilder.setPositiveButton(android.R.string.ok, null);
dialogBuilder.setNegativeButton(android.R.string.cancel, null);
dialogBuilder.show();

As the methods of the class MaterialDialogBuilder, which is provided by this library, is nearly identical to the API of the Android SDK's class AlertDialog.Builder, the practices, which are described in this section of the Android developer guide, do also work together with this library.

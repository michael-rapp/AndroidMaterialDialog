1 Implementation details

The builder MaterialDialogBuilder, which is provided by this library is extended from the Android SDK's built-in class AlertDialog.Builder and therefore provides the same functionality as the base class. The builder creates an instance of the class AlertDialog, whose appearance is designed according to the Android 5.0 "Material Desing" guidelines, which can be found here. Such a dialog can contain a title, an icon, a message and up to three buttons. All of these properties are optional. Furthermore, the dialog can be used to display a list view, which contains a specific set of list items, using a specific choice mode. It is also possible to set custom views for the dialog's title or content in order to realise a "Material Design"-styled custom dialog. How to create dialogs using these different properties will be discussed in the following sections.

So far, the following methods are not implemented yet and therefore calling these methods will cause a OperationNotSupportedException to be thrown:

setMultiChoiceItems(Cursor, String, String, OnMultiChoiceClickListener):MaterialDialogBuilder

setSingleChoiceItems(Cursor, int, String, OnClickListener):MaterialDialogBuilder

setCursor(Cursor, OnClickListener, String):MaterialDialogBuilder

2 Specifying a title, icon and message

In order to create a dialog, an instance of the class MaterialDialogBuilder has to be created at first. The builder therefore provides the constructor MaterialDialogBuilder(Context), which expects an instance of the class Context as a parameter.

To specify the title of the dialog, which should be created by the builder, the following two methods can be used. One of them allows to pass the title as an instance of the class CharSequence, the other one expects the resource ID of a string resource as a parameter.

setTitle(CharSequence):MaterialDialogBuilder

setTitle(int):MaterialDialogBuilder

Specifying the message of a dialog is very similar to specifying the dialog's title. The following methods, which either allow to pass an instance of the class CharSequence or the resource ID of a string resource, can be used.

setMessage(CharSequence):MaterialDialogBuilder

setMessage(int):MaterialDialogBuilder

To show an icon to the left of the dialog's title, the following three methods can be used. The icon is also shown if no title is specified.

setIcon(Drawable):MaterialDialogBuilder

setIcon(int):MaterialDialogBuilder

setIconAttribute(int):MaterialDialogBuilder

The setIcon-methods show above allow to specify the icon by either passing an instance of the class Drawable or the resource ID of a drawable resource as a parameter. The setIconAttribute-method allows to pass the attribute of theme, which is currently applied on the builder's context, which specifies the drawable. An example of such an attribute is android.R.attr.alertDialogIcon.

3 Specifying the dialog's buttons

As already mentioned, the a dialog, which is created by a MaterialDialogBuilder may contain up to three buttons. These buttons, which are called "negative button", "neutral button" and "positive button" are added to the dialog in this particular order from left to right. For adding each of these buttons to the dialog, two dedicated methods exist. One of them allows to specify the button's text as an instance of the class CharSequence, the other one allows to pass the resource ID of a string resource. All of these methods are listed below:

setNegativeButton(CharSequence, OnClickListener):MaterialDialogBuilder

setNegativeButton(int, OnClickListener):MaterialDialogBuilder

setNeutralButton(CharSequence, OnClickListener):MaterialDialogBuilder

setNeutralButton(int, OnClickListener):MaterialDialogBuilder

setPositiveButton(CharSequence, OnClickListener):MaterialDialogBuilder

setPositiveeButton(int, OnClickListener):MaterialDialogBuilder

The second parameter of all methods shown above allows to pass a listener, which therefore must implement the interface DialogInterface.OnClickListener, to the button. The listener is invoked when the user closes the dialog by clicking the appropriate button. Passing a null reference instead of a listener implementation, does not cause the app to crash and the dialog will be closed anyways, when the button is clicked.

By default the dialog's buttons are align horizontally, but as the "Material Design" guidelines introduced an alternative way of aligning a dialog's buttons, the builder's method stackButtons(boolean):MaterialDialogBuilder can be used to align the buttons vertically. 5 Use the dialog to show list items

A dialog, which is created by the class MaterialDialogBuilder can also be used to show list items. Three different choice modes are available. Each one of them will be discussed in one of the following subsections.

5.1 No choice mode

To show list views within the dialog using no choice mode, which means that the items are not selectable, can be established by using one of the following methods:

setItems(CharSequence[], OnClickListener):MaterialDialogBuilder

setItems(int, OnClickListener):MaterialDialogBuilder

setAdapter(ListAdapter, OnClickListener):MaterialDialogBuilder

The setItems-methods mentioned above can be used to specify the list items as an array of CharSequence objects or by passing the resource ID of an array resource. When using these methods, the builder will create an adapter to manage the items by itself. Alternatively, the setAdapter-method allows to pass such an adapter to the builder. Such an adapter has to take care of storing and visualizing the items. Furthermore, all of the methods shown above allows to pass a listener, which has to implement the interface DialogInterface.OnClickListener, to the builder. Such a listener will be notified, when the user clicks one of the list items. Clicking a list item when using no choice mode will cause the dialog to be closed. 5.2 Single choice mode

To show selectable list items, of whom only one item can be selected at once, the following methods can be used:

setSingleChoiceItems(CharSequence[], int, OnClickListener):MaterialDialogBuilder

setSingleChoiceItems(int, int, OnClickListener):MaterialDialogBuilder

setSingleChoiceItems(ListAdapter, int, OnClickListener):MaterialDialogBuilder

These methods are very similar to the methods discussed in the previous section, but all of them expect an additional Integer value, which specifies the position of the item, which should be selected by default, as a parameter. If this value is set to -1, no item will be selected by default. The semantic of the other parameters is equal to the semantics of the previously discussed method parameters. When a list item is selected while using single choice mode, the dialog will not be closed. The following screenshot shows a dialog, which contains list views using single choice mode:

5.3 Multi choice mode

The third possibility to show list items within the dialog is to use the multi choice mode, which means, that multiple items can be selected at once. This can be established by using one of the following methods:

setMultiChoiceItems(CharSequence[], boolean[], OnMultiChoiceClickListener):MaterialDialogBuilder

setMultiChoiceItems(int, boolean[], OnMultiChoiceClickListener):MaterialDialogBuilder

These methods are slightly different from the already discussed ones, as the listener, which can be passed to be notified, when a list item's selection changes, has to implement the interface DialogInterface.OnMultiChoiceClickListener. Also, the items, which should be selected by default are specified by passing a Boolean array. The array's length must be equal to the number of items, as the Boolean at a specific index indicates, whether the list item at the according position should be selected, or not. Like when using the single choice mode, the dialog is not closed when a list item is selected while using multi choice mode. 6 Using custom views

If the class MaterialDialogBuilder should be used to create a "Material Design"-styled custom dialog, the following methods can be used to replace the dialog's child views, which are used to show its title or its content (the ListView, which is used to show the list items).

setTitleView(int):MaterialDialogBuilder

setView(int):MaterialDialogBuilder

setView(View):MaterialDialogBuilder

The setTitleView-method causes the child view, which is used to show the dialog's title and icon, to be replaced by a specific custom view. If the custom view contains a TextView with the ID android.R.id.title, the dialog's title will be shown by using this view. Otherwise, the dialog's title will not be shown and the custom can be used for any other purpose. When the setView methods are used to specify a custom content view by passing the view itself or the resource ID of the appropriate layout resource, the ListView, which is used to show the dialog's list items, will be replaced. If the custom view contains a ListView with the ID android.R.id.list, this view will be used to show the dialog's list items. Otherwise, the items can not be shown. 7 Creating and showing the dialog

Once an instance of the class MaterialDialogBuilder has been configured by using the setter methods discussed in the previous sections of this documentation, the builder's method create():AlertDialog can be used to create the dialog as an instance of the class AlertDialog. Such a dialog instance can be shown by calling its show-method. A shorter way of showing the dialog, which is created by a MaterialDialogBuilder, is to call the builder's show-method. This method implicitly calls the create-method to create the dialog, as well as the dialog's show-method to show the dialog on the screen.

8 Validating the dialog's content

It is also possible to validate a dialog's content when its positive button is clicked by the user. Therfore the interface Validator has to be implemented. It defines the method validate():boolean, which has to be overridden in order to return, whether the dialog's content is valid, or not. Multiple instances of this interface can be added to an instance of the class MaterialDialogBuilder by using its method addValidator(Validator):void. Previously added instances can be removed by using the method removeValidator(Validator):void afterwards. When the dialog's positive button is clicked by the user, all added validators will be executed. If one of them returns false from its validate-method, the dialog will not be dismissed. When the negative or neutral button is clicked instead, no validation will be performed.
